Flexbox

  • Flex Direction: Define la dirección del flujo principal de los elementos flexibles (horizontal o vertical).
  • Flex Wrap: Controla si los elementos deben ajustarse en una sola línea o en múltiples líneas.
  • Flex Flow: Combinación de flex-direction y flex-wrap.
  • Justify Content: Alinea los elementos a lo largo del eje principal (inicio, centro, final, espacio entre elementos).
  • Gap: Define el espacio entre los elementos flexibles.
  • Align Items: Alinea los elementos a lo largo del eje transversal (centro, inicio, final).
  • Order: Establece el orden en que se muestran los elementos dentro del contenedor flexible.
  • Flex Grow: Controla cuánto crece un elemento en comparación con los demás elementos.
  • Flex Shrink: Define cómo se reduce el tamaño de un elemento en relación con los demás cuando no hay espacio suficiente.
  • Flex Basis: Establece el tamaño inicial del elemento antes de que se distribuya el espacio adicional.
  • Flex: Shorthand que combina flex-grow, flex-shrink y flex-basis.
  • Align Self: Permite sobrescribir align-items para un elemento específico.

SVG

Origen: SVG (Scalable Vector Graphics) es un formato gráfico basado en XML para definir gráficos vectoriales bidimensionales. Fue desarrollado para usarse en la web y ofrece una forma de representar imágenes que se pueden escalar sin pérdida de calidad.

Ventajas:

  • Escalabilidad sin pérdida de calidad.
  • Menor tamaño de archivo en comparación con gráficos rasterizados para imágenes simples.
  • Interactividad y animación utilizando CSS y JavaScript.
  • Compatible con SEO al poder contener texto dentro de la imagen.

Información general: SVG se utiliza en múltiples aplicaciones, desde íconos y logotipos hasta gráficos complejos. Es ideal para diseños responsivos y permite una personalización fácil.

:root

:root es el selector de CSS que representa el nivel más alto del documento. Permite definir variables globales que se pueden reutilizar en todo el documento, como colores, fuentes y tamaños.

Ejemplo de uso:

                :root {
                    --primary-color: #333;
                    --font-size-large: 2rem;
                }

Las variables definidas en :root pueden utilizarse en el resto del CSS como var(--primary-color).

Frameworks

Los frameworks de CSS son herramientas que facilitan el desarrollo de interfaces, al ofrecer estructuras de diseño predefinidas y estilos comunes. Ejemplos populares incluyen Bootstrap, Foundation y Bulma, que permiten crear layouts responsivos de forma rápida sin necesidad de escribir CSS desde cero.

Unidades Responsivas

Las unidades responsivas permiten adaptar el diseño a diferentes tamaños de pantalla:

  • vw: Unidad que representa el 1% del ancho de la ventana.
  • vh: Unidad que representa el 1% de la altura de la ventana.
  • rem: Unidad relativa basada en el tamaño de la fuente del elemento raíz (generalmente html).
  • em: Unidad relativa basada en el tamaño de la fuente del elemento padre inmediato.

Estas unidades son fundamentales para crear un diseño adaptable a diferentes dispositivos.

Mobile First

El enfoque de Mobile First consiste en diseñar una página web para dispositivos móviles como prioridad y luego adaptarla a pantallas más grandes. Esto garantiza que el sitio sea eficiente y accesible en dispositivos con menos capacidad y tamaño de pantalla.

Se suele combinar con media queries, donde se definen primero los estilos para móviles y luego se amplían para pantallas más grandes.

Módulo Grid

CSS Grid es un sistema de diseño de dos dimensiones que permite crear layouts complejos. A diferencia de Flexbox, que trabaja en una sola dimensión, Grid permite organizar elementos en filas y columnas.

Propiedades importantes:

  • grid-template-columns: Define el número y tamaño de las columnas.
  • grid-template-rows: Define el número y tamaño de las filas.
  • grid-gap: Espacio entre las celdas de la cuadrícula.
  • grid-area: Permite que los elementos ocupen varias celdas.

@supports

@supports es una regla de CSS que permite aplicar estilos solo si el navegador es compatible con una propiedad o valor específico. Esto permite crear estilos alternativos o "fallbacks" para navegadores que no soportan algunas propiedades avanzadas.

Ejemplo:

                @supports (display: grid) {
                    .container {
                        display: grid;
                    }
                }

En el ejemplo, se aplica display: grid solo si el navegador es compatible con esta propiedad.
